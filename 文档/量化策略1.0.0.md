# 🎯 量化交易员专用指标体系
## 精准识别拉盘/砸盘，捕捉买卖点

作为量化交易员，你的目标是**预测价格方向**和**捕捉买卖点**，而不是做市商的被动挂单策略。基于你提供的API，我整理了一套专门用于**识别主力意图**和**预测拉盘/砸盘**的指标体系。

---

## 🚨 一、拉盘/砸盘预警核心指标（5大信号）

### **1. 订单簿失衡度加速变化**
**API来源：** `depth` 连续轮询

```python
# 核心指标：订单簿斜率变化率
bid_slope = 计算买单斜率(最近N档)  # 正值为买盘增加
ask_slope = 计算卖单斜率(最近N档)  # 负值为卖盘减少
slope_divergence = bid_slope + abs(ask_slope)  # 数值越大，失衡越严重

# 加速信号
slope_acceleration = slope_divergence - prev_slope_divergence  # 正值为失衡加剧
```

**拉盘信号：** `bid_slope > 阈值` 且 `ask_slope < -阈值` 且 `slope_acceleration > 0`  
**砸盘信号：** 相反

### **2. 大单吃筹检测**
**API来源：** `aggTrades` 或 `trades`

```python
# 检测连续主动大单
large_trades = [t for t in trades if t.qty > volume_threshold]
consecutive_buy = 连续主动买入(t.isBuyerMaker=False)的大单数量
consecutive_sell = 连续主动卖出(t.isBuyerMaker=True)的大单数量

# 资金流强度
buy_volume_ratio = 主动买入量 / 总成交量  # >0.6 为买方主导
```

**拉盘信号：** `consecutive_buy >= 3` 且 `buy_volume_ratio > 0.7`  
**砸盘信号：** `consecutive_sell >= 3` 且 `buy_volume_ratio < 0.3`

### **3. 价格-成交量背离**
**API来源：** `klines` + `aggTrades`

```python
# 量价背离检测
price_change = (close - open) / open
volume_change = (volume - avg_volume) / avg_volume

# 主力是否在偷偷出货/吸筹
if price_change > 0 and volume_change < -0.3:
    signal = "量缩上涨，主力可能拉高出货"  # 危险！
    
if price_change < 0 and volume_change > 0.3:
    signal = "量增下跌，可能主力砸盘吸筹"  # 观察企稳后可抄底
```

### **4. 订单簿恢复力异常**
**API来源：** `depth` 高频轮询

```python
# 测量大单被吃后的恢复速度
初始深度 = 获取当前深度
模拟吃单 = 吃掉top N档
time.sleep(0.5)
新深度 = 获取当前深度
恢复率 = 新深度 / 初始深度

# 恢复力异常
if 恢复率 < 0.5:  # 深度恢复慢，真实买盘不足
    pump_signal = "假拉升，容易被反杀"
    
if 恢复率 > 1.5:  # 深度反而增加，有人在托底
    dump_signal = "假砸盘，下方承接强"
```

### **5. 微价格发现**
**API来源：** `depth` + `trades`

```python
# 计算加权中间价
weighted_mid = (bid_price1*ask_qty1 + ask_price1*bid_qty1) / (bid_qty1 + ask_qty1)

# 与实际成交价的偏离
price_divergence = (last_trade_price - weighted_mid) / weighted_mid

# 偏离信号
if price_divergence > 0.001:  # 成交价高于加权中间价0.1%
    signal = "价格被拉高，但订单簿不支持，随时回调"
    
if price_divergence < -0.001:
    signal = "价格被砸低，但订单簿有支撑，可能反弹"
```

## 🎯 二、精准买卖点指标体系

### **1. 订单流失衡速度**
**API来源：** `aggTrades`

```python
# 计算每秒净流量
净流量 = 主动买入量 - 主动卖出量
失衡速度 = 净流量变化率 / 时间

# 买卖点信号
if 失衡速度 > 阈值 and 净流量 > 0:
    entry_point = "加速买入，追涨信号"
    
if 失衡速度 < -阈值 and 净流量 < 0:
    exit_point = "加速卖出，逃顶信号"
```

### **2. 隐含支撑/阻力**
**API来源：** `depth`

```python
# 找到挂单量显著集中的价格区域
bins = 将价格区间分成100个bins
for 每个bin:
    累积挂单量 = sum(该bin内的所有挂单)
    
# 支撑位 = 买盘量最大的价格区域
# 阻力位 = 卖盘量最大的价格区域

# 突破信号
if 价格突破阻力位且阻力位挂单被快速吃掉:
    breakout_signal = "真突破，跟进"
    
if 价格接近阻力位但买盘衰竭:
    rejection_signal = "假突破，反向开空"
```

### **3. 波动率收缩突破**
**API来源：** `klines`

```python
# 计算近期波动率
最近20根K线的ATR = 平均真实波幅
近期波动率 = ATR / 价格

# Bollinger Band宽度
bb_width = (上轨 - 下轨) / 中轨

# 收缩突破信号
if bb_width < 历史分位数0.2:  # 极度收缩
    if 成交量突然放大:
        breakout_direction = "突破方向 = 成交量主导方向"
```

### **4. 订单簿失衡拐点**
**API来源：** `depth` 实时监控

```python
# 监控买卖失衡度的变化趋势
失衡度 = (总买盘 - 总卖盘) / (总买盘 + 总卖盘)
失衡度_MA5 = 移动平均(失衡度, 5)

# 拐点信号
if 失衡度 上穿 失衡度_MA5 and 失衡度 < -0.2:
    reversal_signal = "极度超卖后买盘开始恢复，抄底时机"
    
if 失衡度 下穿 失衡度_MA5 and 失衡度 > 0.2:
    reversal_signal = "极度超买后卖盘开始出现，逃顶时机"
```

## 🚀 三、拉盘/砸盘概率预测模型

### **特征工程（从API提取）**

```python
特征 = {
    # 从depth
    'bid_ask_ratio': 总买盘 / 总卖盘,
    'top_bid_concentration': 最大买单 / 总买盘,  # 是否有托单
    'top_ask_concentration': 最大卖单 / 总卖盘,  # 是否有压单
    'spread_bps': (ask1 - bid1) / ((ask1 + bid1) / 2) * 10000,
    'depth_skewness': 买盘斜率 / 卖盘斜率,
    
    # 从trades/aggTrades
    'buy_sell_ratio': 主动买入量 / 主动卖出量,
    'large_trade_count': 大单数量,
    'taker_volume_ratio': 主动成交量 / 总成交量,
    'volume_acceleration': 成交量变化率,
    
    # 从klines
    'price_position': (当前价 - 最低价) / (最高价 - 最低价),  # 在当日区间的位置
    'volume_ratio': 当前成交量 / 平均成交量,
    'volatility': 收益率标准差,
    
    # 复合特征
    'pump_score': (主动买入量 * 大单系数) / (卖盘深度 * 价差),  # 拉升难度
    'dump_score': (主动卖出量 * 大单系数) / (买盘深度 * 价差),  # 砸盘难度
}
```

### **拉盘概率模型**

```python
拉盘概率 = (
    0.3 * (主动买入量 > 主动卖出量 * 2) +
    0.2 * (买单斜率 > 阈值) +
    0.2 * (大单连续买入次数 >= 3) +
    0.15 * (卖盘深度 < 平均深度 * 0.7) +  # 卖盘薄弱
    0.15 * (价格在支撑位附近)
)

砸盘概率 = (
    0.3 * (主动卖出量 > 主动买入量 * 2) +
    0.2 * (卖单斜率 > 阈值) +
    0.2 * (大单连续卖出次数 >= 3) +
    0.15 * (买盘深度 < 平均深度 * 0.7) +  # 买盘薄弱
    0.15 * (价格在阻力位附近)
)
```

## ⏱️ 四、实战交易策略框架

### **策略1：拉盘跟进策略**

```python
入场条件：
1. 主动买入量 > 主动卖出量 * 1.5
2. 买单斜率 > 0 且加速上升
3. 价格突破最近5分钟高点
4. 卖盘深度没有明显增加（没有压单）

止损：跌破入场价 - 2 * ATR
止盈：目标1 = 下一个阻力位，目标2 = 阻力位 + 1*ATR
```

### **策略2：砸盘抄底策略**

```python
入场条件：
1. 价格连续下跌后，主动卖出量开始萎缩
2. 买盘深度开始恢复（有人在托底）
3. 出现放量十字星或长下影线
4. 订单簿失衡度从极值开始回归

止损：跌破近期低点
止盈：目标1 = 20日均线，目标2 = 前高
```

### **策略3：假突破反向策略**

```python
入场条件：
1. 价格快速突破阻力位，但突破时成交量不足
2. 突破后卖盘深度反而增加（压单出现）
3. 主动买入量没有持续
4. 价格回到阻力位下方

方向：做空
止损：突破高点上方 + 1*ATR
止盈：目标1 = 支撑位，目标2 = 突破前低
```

## 📊 五、实战API调用方案

### **高频轮询策略（每秒）**
```python
# 每秒执行一次
depth = client.depth(symbol="BTCUSDT", limit=100)  # 权重5
trades = client.aggTrades(symbol="BTCUSDT", limit=50)  # 权重4
# 总权重9/秒，540/分钟，在限制范围内
```

### **中频分析策略（每5秒）**
```python
# 每5秒执行一次
depth_500 = client.depth(symbol="BTCUSDT", limit=500)  # 权重25
klines_1m = client.klines(symbol="BTCUSDT", interval="1m", limit=20)  # 权重2
ticker = client.ticker_24hr(symbol="BTCUSDT")  # 权重2
# 总权重29/5秒，348/分钟
```

### **低频信号确认（每分钟）**
```python
# 每分钟执行一次
klines_5m = client.klines(symbol="BTCUSDT", interval="5m", limit=100)  # 权重2
historical_trades = client.historicalTrades(symbol="BTCUSDT", limit=1000)  # 权重25
# 用于确认趋势和计算更稳定的指标
```

## 🎯 六、量化回测建议

要验证这些指标的有效性，你需要：

1. **存储原始数据**：depth + trades + klines 到数据库
2. **回测信号**：在历史数据上测试买入/卖出信号
3. **优化参数**：阈值、时间窗口、权重系数
4. **前向测试**：用小资金实盘验证

**关键成功因素**：
- **低延迟**：订单簿数据每100ms都在变化
- **组合信号**：单一信号误报率高，多信号确认
- **动态阈值**：根据市场波动率动态调整参数

---

这套体系专门为你作为**量化交易员**设计，重点关注**价格方向预测**和**买卖点捕捉**，而不是做市商的被动策略。核心思想是：**跟随主力资金流向，识别市场微观结构的异常，在趋势启动点介入。**
